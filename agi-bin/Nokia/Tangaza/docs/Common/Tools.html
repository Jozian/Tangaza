NAME

Nokia::Common::Tools - Utility module

DESCRIPTION

Provides all the utility methods required for performing commonly performed tasks

METHODS

init_tools

Called on startup. Use for any initialization of this module

play_random

To make the system seem more "real", you can have multiple responses that have the same meaning and play them randomly e.g. ok, fine, cool

Args:

$msg: The full filepath to the file to be streamed

$random_key: if you are randomizing "ok" as explained above, the files should be named as okay-01, okay-02 etc. The random key in this case is the unchanging part of the filename i.e. "okay"

dtmf_quick_jump

Ask the user once if he wants to jump somewhere Assumes press 1 to go there

Args:

$jump_fn: The function to jump to/to be called when the user presses 1

$prompt: The prompt to be asking the user whether they want to jump somewhere

get_yes_no_option

Gives the user an option in the form "For yes press 1, for no press 2".

Args:

$prompt: The prompt that the user needs to respond yes or not to

get_large_number

Allows the user to input a large multi-digit number using the phones keypad. but unlike the Nokia::Tools::get_unchecked_large_number manpage, it plays it back to the user for confirmation.

Args:

$prompt: The prompt asking the user to enter a large number

get_unchecked_large_number

Allows the user to input a large multi-digit number using the phones keypad

Args:

$prompt: The prompt asking a user to enter a number

$number_list: Reference to the array that will hold the large number. Each number input is stored as a separate array element

get_small_number

Allows the user to input a single-digit number using the phones keypad but unlike the Tools::get_unchecked_small_number manpage, it plays it back to the user for confirmation.

Args:

$prompt: The prompt asking a user to input a number

$digits: A string of valid digits that the user is allowed to key in

get_unchecked_large_number

Allows the user to input a single-digit number using the phones keypad.

Args:

$prompt: The prompt asking a user to enter a number

$digits: A string of valid digits that the user is allowed to key in

get_dtmf_input

Allows the user to provide input using the phones keypad. It can be numbers or special characters like * or #

Args:

$prompt: The prompt asking the user to key in a number

$digit: A string of the valid set of digits the user is allowed to key in

record_file

Cleanly record a file to the temp filesystem, letting the user play it back and re-record it. Message should be something like: record your X at the beep

Returns the relative pathname of the file if successful (relative to the temp filesystem, which is kept in memory).

Args:

$instructions_prompt: The prompt that

$length_in_seconds: The maximum time the user is given for the recording

$redo_prompt: The prompt asking the user whether they want to re-record

$delete_on_hangup: Delete the file if the user cancels or hangs up (default is no)

unlink_file

Deletes a file from filesystem

Args:

$filename: filepath of file to delete

unlink_tmp_file

Wrapper round the Tools::unlink_file manpage to only delete temporary files generated by the system

Args:

$filename: filepath of file to delete

stream_file

Plays back an audio file stored in the filesystem.

Args:

$prompts: The file to be streamed

$digits: String of digits that the system can respond to as the file is being streamed. Each digit can generate a defined event

say_number

Plays back a number that is in text format, one character at a time.

Args:

$number: The number to be played back

db_connect

Sets up a DB connection using the Perl::DBI manpage

db_disconnect

Terminates the db connection. Always called when the call terminates

get_hash_file

Generates a unique file name in the format [prefix][filename][suffix]

Args:

$prefix: String prefexing the filename

$suffix: A string appended to the filename

$num_digits: Length of the filename to be generated

rnd_alphanum

Generates a random alphanumeric word

Args:

$num_digits: The length of the alpahumeric word to be generated

user_has_hungup

Checks the channel state to determine if the user has hangup. Returns 1 if true.

init_user

Creates a user if he does not exist, else fills `$self` with info about him.

create_user

Creates a new user and returns id of new user

Args:

$desc: Optional secondary information that make up the profile of the user to be created i.e place_id, language_id, phone

get_action

Lookups up the database for the list of allowed actions to determine what action has been performed in the system.

Returns the id of the performed action

Args:

$action: The action name being looked up

get_callcount

Returns the total number of calls that the user has ever made

Args:

$user_id The id of the user whose call count is being determined

get_user_id

Lookups and returns the user id of a user based on their phone number

Args:

$phone: The phone number being looked up

get_user_name

Returns the username/nickname of a user based on their user id

Args:

$user_id: The user id being used to retrieve the username/nickname

set_user_name

Allows the user to set an audio version of their nickname

Args:

$user_id: The user whose username is being set

$prompt: The prompt asking the user to record their name

set_group_name

Allows a group administrator to set an audio version of their group's name

Args:

$group_id: The group whose name is being set

$prompt: A prompt asking the user to record a name for the group

read_config

Reads the configuration file passed in $path and returns a dictionary initialized with the config file's attributes.

Args:

$path: File path to the config file
